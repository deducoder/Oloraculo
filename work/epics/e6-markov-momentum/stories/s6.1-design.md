# Design: S6.1: Markov Predictor Foundation

## Problem
Traditional rolling-average recent form models (like `RecentFormModel.cs`) fail to capture sequential momentum patterns, such as a team's probability of winning after chaining consecutive wins versus breaking a losing streak. We need a mathematical model that represents these sequences as state transitions.

## Value
Unlocks sequence-based momentum metrics that can refine expected goals and final predictions, improving accuracy on closely matched fixtures.

## Approach
We will build a first-order Markov chain predictor class (`MarkovMomentumPredictor.cs`) that implements the `IPredictor` interface. It parses a team's recent match results (wins, draws, losses) into state transition counts and calculates the next match outcome probability.

### Design Decisions

#### Decision 1: 1st-Order Markov Chain state representation
We will represent match outcomes as a 3-state transition model:
- `0` = Win (W)
- `1` = Draw (D)
- `2` = Loss (L)

#### Decision 2: Context-Symmetric Probability Merging
For a match Home vs Away:
- Home transition probability: $H = (P_{home}(W), P_{home}(D), P_{home}(L))$
- Away transition probability: $A = (P_{away}(W), P_{away}(D), P_{away}(L))$
- Combined outcome:
  - $P(Home Win) = \frac{H_W + A_L}{2}$
  - $P(Draw) = \frac{H_D + A_D}{2}$
  - $P(Away Win) = \frac{H_L + A_W}{2}$
- Normalization is applied at the end to ensure they sum to 1.0.

#### Legacy Sweep
Nothing — this is net-new.

#### Drift Risks
No drift risk identified.

### Transition Matrix Calculation Example
Given Home team match outcomes: `W, W, D, L, L, W, D, W, L, W`
- Sequential transitions:
  - `W -> W` (1 time)
  - `W -> D` (2 times)
  - `W -> L` (1 time)
  - `D -> L` (1 time)
  - `D -> W` (1 time)
  - `L -> L` (1 time)
  - `L -> W` (2 times)
- Current state (last match): `W`
- Transitions from `W`: 1 Win, 2 Draw, 1 Loss (Total 4)
- Home team next-state probabilities: Win = 25%, Draw = 50%, Loss = 25%

## Acceptance Criteria

### MUST
- `MarkovMomentumPredictor` must implement the `IPredictor` interface.
- Priority must be set to `3`.
- Model must parse `HomeRecentMatchHistory` and `AwayRecentMatchHistory` into chronological sequences of W-D-L states.
- Model must fall back to uniform probabilities (1/3 each) if match history contains fewer than 5 results for a team.
- Model must fall back to uniform probabilities (1/3 each) if there are no historical transitions starting from the team's current state.

### SHOULD
- Output descriptive drivers and features used/missing in `MatchPrediction`.

### MUST NOT
- Overwrite or mutate the input `MatchContext` object.

## Testing Strategy
- Unit tests in `MarkovMomentumPredictorTests.cs` using `xUnit`.
- Test cases covering:
  - Simple sequence probability calculation.
  - Correct extraction of team perspective (Win for Away = Loss for Home).
  - Fallback logic for small datasets (< 5 matches).
  - Fallback logic for unvisited states.
