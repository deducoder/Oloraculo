# Scope: S6.3: Selector Momentum Bias

## In Scope
- Register `MarkovMomentumPredictor` in `PredictionService.cs`'s list of built predictors.
- Implement Momentum Bias logic inside `FinalPredictionSelector.cs` to resolve close draw predictions.
- Apply a bias weight of `0.15` shifting probability from `Draw` to `HomeWin` or `AwayWin` when the selected predictor's top pick is `Draw` and the Markov prediction has asymmetric momentum.
- Update `MatchPrediction` fields (`Drivers`, `Explanation`, `Sources`) to document when Momentum Bias is applied.
- Unit tests in `Oloraculo.Web.Tests/Predictors/FinalPredictionSelectorTests.cs` (or added to `PredictorTests.cs`) to verify selector logic and bias application.

## Out of Scope
- Adjusting expected goals (already done in S6.2).
- Modifying other predictors.

## Done when
- `PredictionService` includes `MarkovMomentumPredictor` in the ladder.
- `FinalPredictionSelector` applies Momentum Bias when selected top pick is Draw and Markov momentum is asymmetric.
- Explanations and drivers describe the applied Momentum Bias.
- Code compiles and passes all dotnet tests.
