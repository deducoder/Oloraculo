## Tasks

### Task 1: Register Markov predictor and implement Momentum Bias
- **Description:** Register `MarkovMomentumPredictor` in `PredictionService.cs`, implement `TryBuildMomentumBias` and integration in `FinalPredictionSelector.cs`, and add unit tests to `Oloraculo.Web.Tests/Predictors/PredictorTests.cs` using TDD.
- **Files to modify:**
  - Modify: `Oloraculo.Web/Predictors/FinalPredictionSelector.cs`
  - Modify: `Oloraculo.Web/Services/PredictionService.cs`
  - Modify: `Oloraculo.Web.Tests/Predictors/PredictorTests.cs`
- **TDD Steps:**
  1. Write failing tests in `PredictorTests.cs` for:
     - Applying momentum bias towards home win when selected prediction is Draw and Markov home win probability > 0.40 and away win < 0.25.
     - Applying momentum bias towards away win when selected prediction is Draw and Markov away win probability > 0.40 and home win < 0.25.
     - No bias applied when selected prediction is Draw but Markov has no asymmetry or is degraded.
     - Drivers and explanation strings containing the momentum bias details.
  2. Register `MarkovMomentumPredictor` in `PredictionService.cs`.
  3. Implement `TryBuildMomentumBias` and integrate it into `FinalPredictionSelector.Select`.
  4. Ensure all tests pass.
- **AC Reference:** AC-001, AC-002, AC-003, AC-004.
- **Verification:**
  - `rai gate check gate-tests --scope Oloraculo.Web.Tests/Oloraculo.Web.Tests.csproj`

  - `rai gate check gate-types`
- **Size:** S
- **Dependencies:** None

## Risks and Mitigations

| Risk | Likelihood/Impact | Mitigation |
|------|:-----------------:|------------|
| Probability normalization rounding bugs | L/H | Ensure `OutcomeProbabilities.Normalize()` is called after applying the bias. |
| Multiple biases colliding | L/M | Apply the biases sequentially: calculate ranking bias first, and then apply momentum bias on the top pick of that resulting outcome. |
